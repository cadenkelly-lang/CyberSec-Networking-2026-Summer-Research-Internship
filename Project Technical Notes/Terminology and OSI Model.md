Seven) Application layer
	1) Closest to the end user. Data translated from computer readable to human readable. Something like chrome, safari, email application, etc.
	2) Can also identify communication partners, check available resources, and make sure communication is synced.

Six) Presentation layer:
	1) Takes care of getting data ready for the application layer. Two devices may use different methods of encoding data. So, layer 6 turns incoming data into something readable at the application layer.
	2) Encrypting, decryption
	3) Also compresses the data coming fro the application layer before it sends it to layer 5

Five) Session Layer:
	1) Opening and closing network communications between two interacting devices. The session refers to the time between the opening and closing of the interaction.
	2) Ensures open long enough for all data to be sent, then closes it.
	3) Synchronizes data transfer. If large amount of data is sent, it can set up checkpoints to allow for resuming if interruption happens during the data transmission.

Four) Transport Layer:
	1) End-to-end communication between devices. Taking data in session layer and dividing it into pieces called segments. Transport layer on the receiver handles reassembly. Also manages flow and necessary error messages that need to be sent.
	2) Makes sure it isn't being sent too quick for receiver to handle, and it checks to see if the data transmission was done properly. Requests re-transmission if not.
	3) TCP and UDP port numbers work here at layer 4. IP is at layer 3

Three) Network Layer:
	1) Transfer of data when two networks communicating. Must be two separate ones. Divides the segments that come from the transport layer (packets). Division happens on the senders device, reassembled on the receiver's device. Figures out optimal physical path to get data to destination (called routing).

Two) Data Link Layer:
	1) Similar to network layer, but it facilitates data transfer between two devices using the same network. Here, packets are broken into pieces called frames. Handles flow and error control. Transport layer is different in that it handles flow between two different networks talking to each other.
	2) Two sub-layers:
		1) Media Access Control (MAC):
		2) Logical Link Control (LLC):
	3) Most switches are layer 2, but some are layer three if they facilitate communciation between two networks or VLANS (data needs to be routed, which is a layer 3 task)

One) Physical Layer:
	1) Physical equipment that transfers data, like switches and cables. Data is converted from 1s and 0s. Devices must agree on how to distinguish 1s from 0s so digital data can be converted.



## Condensed Table of Network Terms

| Term                                           | Definition                                                                                                                                                                                                                                         |
| ---------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Autonomous System (AS)**                     | A large network under a single administrative authority (such as an ISP or enterprise), identified by an ASN. Each AS controls a specific range of IP addresses and follows a common routing policy. ASes are the building blocks of the Internet. |
| **ARP (Address Resolution Protocol)**          | Maps IPv4 addresses to MAC addresses on a local network. Devices store mappings in an ARP cache; variants include Proxy ARP, Gratuitous ARP, and Reverse ARP. Because ARP lacks authentication, it is vulnerable to ARP spoofing.                  |
| **Binary**                                     | Base-2 numbering system using only 0 and 1. Each bit position represents a power of 2, making binary the foundation of all computer data processing.                                                                                               |
| **Bit**                                        | The smallest unit of digital data, representing either a 0 or a 1.                                                                                                                                                                                 |
| **Byte**                                       | A group of 8 bits, commonly used to represent a character or small unit of data.                                                                                                                                                                   |
| **Broadcast Address**                          | An IPv4 address used to send traffic to every device on a subnet. It is created by setting all host bits to 1 (e.g., 192.168.1.255 for 192.168.1.0/24).                                                                                            |
| **Broadcast Domain**                           | A group of devices that receive the same broadcast traffic. Routers separate broadcast domains, while VLANs can create multiple broadcast domains on the same physical network.                                                                    |
| **Cache**                                      | High-speed temporary storage used to reduce access time for frequently requested data.                                                                                                                                                             |
| **DHCP (Dynamic Host Configuration Protocol)** | Automatically assigns IP addresses and network settings such as subnet masks, gateways, and DNS servers. Uses the DORA process: Discover, Offer, Request, Acknowledge.                                                                             |
| **DNS (Domain Name System)**                   | Translates domain names into IP addresses. DNS resolution typically involves a resolver, root server, TLD server, and authoritative name server, with caching used to improve performance.                                                         |
| **DNS Root Zone**                              | The top of the DNS hierarchy, represented by ".". It contains references to all top-level domains (TLDs) such as .com and .org.                                                                                                                    |
| **Firewall**                                   | Hardware or software that filters network traffic based on rules. Modern firewalls may provide stateful inspection, deep packet inspection, intrusion prevention, application awareness, NAT, and VPN support.                                     |
| **ICMP (Internet Control Message Protocol)**   | Protocol used for network diagnostics and error reporting. Common uses include ping and traceroute; common messages include Echo Request/Reply and Destination Unreachable.                                                                        |
| **IP (Internet Protocol)**                     | Protocol responsible for addressing and routing packets between networks. IP provides logical addresses but does not guarantee delivery, ordering, or reliability.                                                                                 |
| **IP Port**                                    | A numbered endpoint used by applications and services. Common examples include 80 (HTTP), 443 (HTTPS), 22 (SSH), and 53 (DNS).                                                                                                                     |
| **IP Socket**                                  | A combination of an IP address and port number that uniquely identifies a communication endpoint.                                                                                                                                                  |
| **IP Route**                                   | A rule that tells a device where to send traffic. Routes may be static, dynamic, or default, and are stored in routing tables used to determine the next hop.                                                                                      |
| **IPv4 (Internet Protocol Version 4)**         | Uses 32-bit addresses written in dotted-decimal format. Supports about 4.3 billion addresses and commonly relies on NAT to conserve address space.                                                                                                 |
| **IPv6 (Internet Protocol Version 6)**         | Uses 128-bit hexadecimal addresses, providing a vastly larger address space than IPv4. Supports features such as auto-configuration and reduced dependence on NAT.                                                                                 |
| **IPAM (IP Address Management)**               | Software or processes used to track, allocate, and manage IP address space, subnets, DNS records, and DHCP information.                                                                                                                            |
| **Internet**                                   | A global network of interconnected networks that communicate using the TCP/IP protocol suite.                                                                                                                                                      |
| **LAN (Local Area Network)**                   | A network covering a limited area such as a home, office, or campus. LANs commonly use Ethernet or Wi-Fi and allow devices to communicate directly.                                                                                                |
| **MAC Address (Media Access Control Address)** | A unique Layer 2 hardware identifier assigned to a network interface. MAC addresses are typically 48 bits long and are used for local network communication.                                                                                       |
| **Network Switch**                             | Connects devices within a LAN and forwards traffic based on MAC addresses (Layer 2) or IP addresses (Layer 3). Managed switches support features such as VLANs and traffic control.                                                                |
| **Network**                                    | A collection of connected devices that share data and resources through wired or wireless communication.                                                                                                                                           |
| **NTP (Network Time Protocol)**                | Synchronizes time across networked devices. NTP uses a hierarchy of servers called strata, with lower strata providing more authoritative time sources.                                                                                            |
| **OPNsense**                                   | Open-source firewall and routing platform based on FreeBSD. Provides stateful firewalling, routing, VPN, and security management features.                                                                                                         |
| **Proxmox**                                    | Open-source virtualization platform used to manage virtual machines and containers.                                                                                                                                                                |
| **SDDC (Software-Defined Data Center)**        | Software-Defined Data Center; virtualizes compute, storage, networking, and security resources so they can be managed through software.                                                                                                            |
| **Segmented Network**                          | A network divided into smaller logical or physical sections to improve security, performance, and traffic control. Common methods include VLANs, routers, and firewalls.                                                                           |
| **Subnet**                                     | A logical subdivision of an IP network. Each subnet contains a network address, host addresses, and (in IPv4) a broadcast address.                                                                                                                 |
| **Subnet Mask**                                | Defines which bits of an IPv4 address represent the network and which represent the host. Often written in CIDR notation (e.g., /24).                                                                                                              |
| **TCP (Transmission Control Protocol)**        | Reliable, connection-oriented transport protocol that uses a three-way handshake and guarantees ordered delivery through acknowledgments and retransmissions.                                                                                      |
| **UDP (User Datagram Protocol)**               | Connectionless transport protocol that prioritizes speed over reliability. Commonly used for DNS, streaming, VoIP, and online gaming.                                                                                                              |
| **Virtual Machine (VM)**                       | Software-based computer that runs its own operating system and applications using resources provided by a hypervisor.                                                                                                                              |
| **VLAN (Virtual Local Area Network)**          | A Layer 2 technology that logically separates devices into different virtual networks without requiring separate physical infrastructure.                                                                                                          |
| **WAN (Wide Area Network)**                    | A network that connects multiple LANs across large geographic areas using technologies such as leased lines, fiber, satellite links, or VPNs.                                                                                                      |
